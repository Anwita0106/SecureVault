# API Documentation

The full interactive API documentation is auto-generated by FastAPI and available at:

- **Swagger UI**: `http://localhost:8000/api/docs`
- **ReDoc**: `http://localhost:8000/api/redoc`
- **OpenAPI JSON**: `http://localhost:8000/api/openapi.json`

When running via Docker Compose with the Nginx frontend, these are also reachable
through the proxy at `http://localhost/api/docs` etc.

## Authentication Flow

1. `POST /api/v1/auth/register` - create an account (default role: `user`)
2. `POST /api/v1/auth/login` - returns `access_token` (30 min) + `refresh_token` (7 days)
3. Include `Authorization: Bearer <access_token>` on all subsequent requests
4. When the access token expires, `POST /api/v1/auth/refresh` with the refresh token
   to get a new pair (the frontend Axios client does this automatically via an
   interceptor)

## Example: Upload and Share a File

```bash
# 1. Login
TOKEN=$(curl -s -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@securevault.local","password":"Admin@SecureVault2024!"}' \
  | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 2. Upload a file
curl -X POST http://localhost:8000/api/v1/files/upload \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@./report.pdf" \
  -F "description=Quarterly report"

# 3. Create a share link (file_id=1, expires in 24h, max 5 downloads)
curl -X POST http://localhost:8000/api/v1/files/1/share \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "file_id": 1,
    "can_download": true,
    "can_view": true,
    "expires_hours": 24,
    "max_downloads": 5
  }'

# Response includes a "token" - share via:
# http://localhost/share/<token>
```

## Error Response Format

All errors follow FastAPI's standard format:

```json
{
  "detail": "Human-readable error message"
}
```

Validation errors (HTTP 422) return a list:

```json
{
  "detail": [
    {
      "loc": ["body", "password"],
      "msg": "Password must contain at least one uppercase letter",
      "type": "value_error"
    }
  ]
}
```

## Rate Limiting

The platform enforces request rate limits (`RATE_LIMIT_PER_MINUTE`, default 60/min
per IP). Exceeding this returns HTTP 429.
